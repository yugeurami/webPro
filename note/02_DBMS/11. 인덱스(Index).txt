[인덱스(Index)]
조회를 빠르게 하기 위해 사용
CREATE INDEX index_name ON table_name (column_name);

<장점>
검색속도가 빨라진다
시스쳄에 걸리는 부하를 줄여 시스템 전체 성능을 향상시킴

<단점>
추가적인 공간이 필요
인덱스 생성에 시간이 소요
데이터 변경 작업(INSERT/UPDATE/DELETE)시 오히려 성능이 저하

<인덱스를 사용해야하는 경우>
테이블의 행의 수가 많을 때
WHERE 문에 해당 컬럼이 많이 사용될 때
검색 결과가 전체 데이터에 2~4% 정도 일 때
JOIN에 자주 사용되는 컬럼이나 NULL을 포함하는 경우
