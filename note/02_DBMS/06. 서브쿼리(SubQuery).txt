[서브쿼리(SUB QUERY)﻿]
﻿서브 쿼리는 하나의 SQL 문장의 절 안에 포함된 또 하나의 SELECT 문장을 의미한다
서브 쿼리를 포함하고 있는 쿼리문을 메인 쿼리, 포함된 또 하나의 쿼리를 서브 쿼리라고 한다
서브 쿼리는 비교 연산자의 오른쪽에 기술하고 반드시 괄호로 둘러 쌓여있어야한다.
서브 쿼리는 메인 쿼리가 실행되기 이전에 한번만 실행된다.
SELECT, FROM, WHERE 절에 모두 들어갈 수 있다.

﻿﻿
<서브쿼리의 종류>
1. 단일행 서브쿼리
수행 결과가 오직 하나의 행(row)만을 반환하는 서브 쿼리를 말한다
WHERE 절에서 단일 행 비교 연산자(=, >, >=, <, <=, !=)를 사용할 수 있다
SELECT 절의 서브쿼리는 1행 1열만 들어간다

﻿
2. 다중행 서브쿼리
수행 결과가 2개 이상의 행(row)을 반환하는 서브 쿼리를 말한다
단일행 연산자를 사용할 수 없고 다중행 연산자와 함께 사용해야한다
다중행 서브쿼리는 단독적으로 실행이 안될 수도 있다

IN		서브쿼리 결과 중 하나라도 일치하면 참
ANY, SOME	서브쿼리 결과가 하나 이상 만족하면 참
ALL		서브쿼리 결과를 모두 만족해야 참
EXISTS		서브쿼리 결과가 존재하면 참
