
[예외처리]
프로그램에서 문제가 발생될 만한 곳을 예상하여 사전에 "문제가 발생하면 이렇게 해라"라고 프로그래밍 하는 것


<필요성>
어느 한 부분에서 예외가 발생하더라도 계속해서 프로그램이 동작되도록 하기위해서 예외처리를 해야한다.
예외처리는 보험이다


[try~catch]
try {
	// 오류가 발생할 가능성이 있는 명령문들
}catch(exceptionType 변수명) { // 여러 개의 catch가 오면 위에서 부터 찾아나감
     // 예외 발생 시 실행하는 명령문
} finally {
	 // 예외 발생 여부와 상관없이 맨 마지막에 실행할 명령문
}


[throws]
예외 처리를 하지않고 이 메소드를 불러오는 곳에서 처리하라고 예외를 던져주는 것
throws를 사용할 경우 어느 한 곳에서는 반드시 try~catch 처리를 해줘야함
Main class에서 throws를 하는 것은 지양
